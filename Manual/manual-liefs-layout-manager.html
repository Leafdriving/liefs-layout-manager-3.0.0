Builder Application

Liefs Layout Manager is a Javascript Framework (writen in Typescript) that manages the position of 'Div' Elements on a Page. The position of elements on the page is determined by using elements (Classes) of the framework, called by simple functions. In essence, It ignores the inheritance of 'div' objects, and places them all on the first level of the dom, using a coordinate system. (x, y, width, height) and assigns these values in the 'style' tag of the element. Most of the 'rules' of the framework are intuitive, and easy to learn by Novice, and Experienced Programmers Alike. This Manual is written under this framework, and includes most of the features. For example, try re-sizing this window to see how the layout changes to fit all sizes. Try the Dragbar, and tree system, all managed by this framework. There are TONS of examples included here to see each feature working in it's simpleist, to most difficult form.

This framework only has 'divs' in the DOM that are viewed! If a 'div' isn't currently desired on the screen, then it is REMOVED from the dom, and re-instroduces when required. For example, THIS MANUAL is loaded as one page but if you check the dom (Chrome Dev Tools F12) you will see it is CLEAN - with only the 'div's viewed existing in the DOM.

Liefs Layout Manager is a CLIENT SIDE framework. It simply organizes 'div's on the screen. What you put in the 'div's is up to you. It could be used for:

  • Making a RESPOSIVE Web App
  • Or Even a simple Web Page
  • Making a Web Application
  • UI for Electron projects
  • In conjunction with other framworks

This framework is designed to work WITH your code, and only enhances your abilities. All other layout systems work in conjunction with this one. The only restriction is you can't use the 'style' inline attribute for liefs-layout-manager divs. (but you can for child divs) this is because the mapping system overrides these values with "left: ; top: ; width: ; height: ; zindex: " attributes.

Liefs Layout Manager is basically an Object Tree, just like all file systems.

Terms

  • DisplayCell : All nodes of the tree are contained within a DisplayCell
  • DisplayGroup : Holds an array of DisplayCells (Children) within a DisplayCell
  • Handler : Holds the "root" DisplayCell, and renders the result to the screen

All Objects in Liefs-layout-manager MUST have a label. If one is not specified, it will be automaticaly Generated.

Both CSS and 'Div' elements can OPTIONALLY be added in either the HTML file, OR the javascript part/file.

Liefs-Layout-Manager is a frameword of nested (Class) Objects.

In order to make Object Creation easier, I have added a lot of "Helper" functions whose only purpose is to help with Object Creation. All "Pure Functions", which have nothing to do with object creations are located in the classes "pf" and "mf".

For Example, I() is defined as:

function I(...Arguments:any) /*: DisplayCell*/ {
  return new DisplayCell( new HtmlBlock(...Arguments) )
}

So the code:

I("SomeLabel", "Some Inner Html")

could also be written as:

new DisplayCell( new HtmlBlock("SomeLabel", "Some Inner Html") )

So, function I() supplies the arguments to an htmlBlock, wrapped in a DisplayCell

(almost) All liefs-layout-manager functions and Objects use "Argument By Type, then position" rather than (default) Position

To show this in an Example:

somefunction(1, "Two", new Three(), function(four){}, true, 5, "six")

and

somefunction(true, new Three(), 1, 5, "Two", "six", function(four){})

Are EQUIVELENT, and would both be intrepreded as:

{"string" : ["Two", "Six"], // first "string" argument is "Two", 2nd is "Six"
"number" : [1, 5], // first "number" is 1, 2nd is 5
"Three" : [ ThreeInstance  ], // first type (object) "Three" is ThreeInstance
"function" : [ function(four){} ], // first "function" is function(four){}
"boolean" : [ true ], // first "boolean" is true
}

When coding liefs-layout-manager in javascript/Typescript, you will "mostly" be using helper functions created with the purpose of "easy" object creation.

When coding liefs-layout-manager in javascript/Typescript, all functions and objects simply set parameters of some sort. NOTHING will happen until you initiate a Handler. This is done with the handler Object (function). A Handler function H() simply creates a new Handler Object. 

function H(...Arguments/*: any*/) /*: Handler */ {
  return new Handler(...Arguments)
}

By Default, a Handler is "On" (rendering, and watching for events) once this function is completed. to "Register" a handler, but keep it "off" (not rendering, and watching for events), simply include a "false" as one of the arguments:

H("Some", h("arguments"), false, "other Arguments")

Every Example HAS TO HAVE a handler, so... look at ANY other example as well!

A Handler Instance has the following properties:

label /*:string;*/                   // The name you give to the handler
rootCell /*:DisplayCell*/            // any Display Cell (ie I(), h(), v() ...) Which acts as the Root.
coord /*:Coord;*/                    // (optional) define x,y,width,height of Handler with a Coord Object
cssString /*: string;*/              //
handlerMargin /*: number;*/          // Number of pixels to leave "Blank" around the outside (Usuallu Screen size)
addThisHandlerToStack /*: boolean;*/ // Default: true -> Show right Away - false -> I will display it later
preRenderCallback /*: Function; */   // function to call before Handler is Rendered: function(handlerInstance){}
postRenderCallback /*: Function;*/   // function to call after Handler is Rendered: function(handlerInstance){}

These can be included in the arguments according to the typemap:

{ string : ["label"],
  number : ["handlerMargin"],
  Coord: ["coord"],
  function: ["preRenderCallback", "postRenderCallback"],
  boolean: ["addThisHandlerToStack"]}

So for example, the following Code:

var handler = H("MyHandler",
  I("OneCellOnly","innerHtml of Cell"),
  10,
  new Coord(0,0,50,50),
  false,
  function(handlerInstance) { console.log("Handler Is about to be calculated") },
  function(handlerInstance) { console.log("Handler has been created") }
);

Would assign variable handler to a new Handler() Object with the following properties:

handler.label = "MyHandler"
handler.rootCell = // (A DisplayCell Object with label "OneCellOnly")
handler.coord = // (A Coord Object with x=0, y=0, width=50, height=50)
handler.handlerMargin = 10 // leave a 10 pixel border on all 4 sides
handler.addThisHandlerToStack = false // so define, but do not show right away
handler.preRenderCallback = function(handlerInstance) { console.log("Handler Is about to be calculated") }
handler.postRenderCallback =function(handlerInstance) { console.log("Handler has been created") }

See Handler Object for Static Properties.

All placement of "Where" Components (Divs) go on a page, are determined by "DisplayGroups". From the Root Cell (Most likely, the screen (Size) ), to the smallest cells, all positioning is determined by "DisplayGroups". h() and v() are simply wrapper functions for a DisplayGroup.

function h(...Arguments /*: any*/){
  return new DisplayCell( new DisplayGroup(...Arguments) )
}
function v(...Arguments /*: any*/){
  return new DisplayCell( new DisplayGroup(false, ...Arguments) );
}

Both h() and v() return a DisplayGroup wrapped in a DisplayCell. h() is used for cells placed horizontally, and v() is for cells places vertically.

A DisplayGroup Instance has the following properties:

label /*:string;*/             //
ishor /*:boolean;*/            // The name you give to the DisplayGroup
cellArray /*: DisplayCell[] */ // The Array of DisplayCells contained within this DisplayGroup
coord /*: Coord;*/             // This is handled interally - it is unlikely you would set it.
htmlBlock /*: HtmlBlock */     // A Background (Usually a color/texture) that is UNDER the DisplayCells in the Group
marginHor /*:number; */        // The horizontal space BETWEEN cells (no margins at the ends)
marginVer /*:number; */        // The vertical space BETWEEN cells (no margins at the ends)
dim /*:string; */              // this is a "Dim" value (string) ie "120px" (fixed size) or "50%"

These can be included in the arguments according to the typemap:

{ string : ["label"],
  boolean : ["ishor"],                 // note: h() means "true", and v() means false
  number: ["marginHor", "marginVer"],
  dim : ["dim"],
  DisplayCell: //all provided
}

So for example, the following Code:

var horDGroup = h("MyHorizontal",
  I("leftSize","innerHtml of left Div"),
  I("rightSize","innerHtml of right Div"),
  "15px",
  5,
  new htmlBlock("myBackgroun","", css("bgCyan","background-color: cyan;"))
)

Would assign variable horDGroup to a new DisplayGroup() Object with the following properties:

horDGroup.label = "MyHorizontal"
horDGroup.ishor = true // This was set true, because we used h() not v()
horDGroup.cellArray = [I("leftSize","innerHtml of left Div"),I("rightSize","innerHtml of right Div")]
horDGroup.htmlBlock = new htmlBlock("myBackgroun","", css("bgCyan","background-color: cyan;")) // make background color cyan
horDGroup.marginHor = 5 // there are now 5 blank pixels between Cells
horDGroup.dim = "15px" // when this DisplayGroup is displayed, it should be 15 pixels in size (Parent Dependent)
